A PowerShell script that automates ad-blocking and malware protection by managing the Windows hosts file. Downloads blocking rules from someonewhocares.org while preserving your custom host entries.

‚ú® Features

  • Automated Ad-Blocking: Downloads and applies thousands of ad/malware blocking rules
  • Preserves Custom Entries: Automatically detects and preserves your custom host entries
  • Automatic Backups: Creates timestamped backups before any changes
  • Smart Updates: Detects if hosts file is already up-to-date to avoid unnecessary changes
  • Backup Management: Keeps last 10 backups, automatically removes older ones
  • DNS Cache Flushing: Automatically flushes DNS cache to apply changes immediately
  • Comprehensive Logging: Follows myTech.Today logging standards with monthly rotation
  • WhatIf Support: Preview changes before applying them
  • Restore Capability: Easy restoration from any backup

üìñ Usage

Basic Update

.\hosts.ps1

Downloads latest blocking rules and updates the hosts file while preserving custom entries.

Preview Changes (WhatIf)

.\hosts.ps1 -WhatIf

Shows what changes would be made without actually applying them.

Force Update

.\hosts.ps1 -Force

Forces update even if the hosts file is already up-to-date. Skips confirmation prompts.

Backup Only

.\hosts.ps1 -BackupOnly

Creates a backup of the current hosts file without making any changes.

Restore from Latest Backup

.\hosts.ps1 -RestoreBackup

Restores the hosts file from the most recent backup.

Restore from Specific Backup

.\hosts.ps1 -RestoreBackup -BackupPath "C:\Users\YourName\myTech.Today\hosts\backups\hosts.backup.2025-01-21_143022"

Restores the hosts file from a specific backup file.

Skip DNS Flush

.\hosts.ps1 -SkipDNSFlush

Updates the hosts file but skips flushing the DNS cache.

‚öôÔ∏è Parameters

Parameter Type Description
-BackupOnly Switch Creates backup of current hosts file without updating
-RestoreBackup Switch Restores hosts file from most recent backup
-BackupPath String Specific backup file to restore (used with -RestoreBackup)
-SkipDNSFlush Switch Skip DNS cache flush after updating hosts file
-Force Switch Skip confirmation prompts and bypass up-to-date check
-WhatIf Switch Preview changes without applying them

üîç How It Works

  1. Administrator Check: Verifies script is running with admin privileges
  2. Backup Creation: Creates timestamped backup of current hosts file
  3. Download Rules: Downloads latest blocking rules from someonewhocares.org
  4. Up-to-Date Check: Compares downloaded rules with current hosts file
  5. Extract Custom Entries: Identifies and preserves your custom host entries
  6. Merge Content: Combines blocking rules with custom entries
  7. Update Hosts File: Writes merged content to Windows hosts file
  8. Flush DNS Cache: Applies changes immediately by flushing DNS cache
  9. Cleanup: Removes old backups (keeps last 10)

üìù Custom Host Entries

The script automatically preserves custom host entries. Custom entries are identified as:

  • Entries marked with the custom section marker
  • Entries that don't originate from someonewhocares.org
  • User-added IP address mappings
üí° Tip: Your custom entries will be preserved in a dedicated section marked with:
# === Custom Host Entries (Preserved by myTech.Today hosts.ps1) ===

  • Last 10 backups are retained automatically
  • Older backups are automatically removed
  • Backups can be restored at any time

üêõ Troubleshooting

Script Won't Run

  • Ensure PowerShell is running as Administrator
  • Check PowerShell execution policy: Set-ExecutionPolicy RemoteSigned -Scope CurrentUser

Changes Not Taking Effect

  • Manually flush DNS: ipconfig /flushdns
  • Restart your browser
  • Clear browser cache

Restore Not Working

  • Verify backup files exist in backup directory
  • Ensure running as Administrator
  • Check log files for detailed error messages
